⚠️ Critical Operational Notes ⚠️

1. All IP addresses, hostnames, and resource identifiers shown in this guide are illustrative only!

2. Always use the values returned by your own CLI commands or automation outputs - Never copy example values directly into a live environment.

3.  Cloud resources incur costs while provisioned, even when idle - Ensure all resources are cleaned up when you are finished:

3.1 Manual provisioning: Stop or terminate resources using your cloud provider's console or CLI;

3.2 Infrastructure as Code: Destroy all provisioned resources using the same tooling that created them (for example, terraform destroy).

Do not leave infrastructure running unintentionally!

## Implementation Notes

When using the `terraform-aws-modules/iam/aws` Karpenter controller policy, ensure that EC2 instances launched by Karpenter are tagged with:

```yaml
karpenter.sh/discovery: <cluster-name>
```

This tag must be defined in the `EC2NodeClass` under `spec.tags` so that every EC2 instance launched by Karpenter inherits the required discovery tag.

The IAM policy generated by the Terraform module restricts `ec2:TerminateInstances` to instances carrying this tag. Without it, Karpenter can successfully provision worker nodes but cannot terminate them during node consolidation or cluster destruction, leaving orphaned EC2 instances and preventing a clean `terraform destroy`.
